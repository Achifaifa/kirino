Dungeon class. Generates 2D arrays with tiles, forming a dungeon that can be crawled by a player or mob object.

Libraries used:
os, random, player, mob

Functions:

__init__(self,int x,int y):
Class constructor. x and y are the horizontal and vertical size of the dungeon (Array). The minimum size is 40x20.

advmap(self, int x, int y, int xmapsize, int ymapsize):
Creates an advanced map of the filled dungeon of width xmapsize and length ymapsize centered in (x,y). If (x,y) is close to the border of the dungeon array, they will be set so nothing outside the array is displayed.

advmapcoords(self, int x, int y):
Creates an advanced map of size 20x10 (default) centered in the position (x,y). Same restrictions apply.

debug(self):
Checks if the dungeon is correctly built
Returns an integer as error condition code:
EC0: No errors were found (The array meets the requirements to be used).
EC1: There is no entrance or exit.
EC2: The dungeon is below the minimum size, either horizontally or vertically.
EC3: Can't reach the exit from the entrance (planned feature)
EC4: There are halls or rooms unconnected or unreachable (planned feature)

dumpdung(self,int place):
creates a list of all the positions in the dungeon and the terain type in it.
If place is 0, the function dumps the dungeon data into a text file located in ./kirino.dump. If a dump file already exists, it creates a separator and appends a new dump.
If place is not 0, then it dumps on console.

fill(self, player):
Creates a temporary 2D array (dungeon.filled[y][x]), which contains the player marker (8) and the objects overlapped over the dungeon floor tile identifiers.

map(self):
Creates a map on console showing the entire dungeon.

minimap(self, player):
Creates an advanced map of size 20x10 centered in the position of the player object.
